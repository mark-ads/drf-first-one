# Проект на Django REST Framework


# Описание проекта:

Бекенд для проекта менеджмента мероприятий.
Планируйте, создавайте и изменяйте мероприятия.

---

## Реализованный функционал:

- CRUD для мероприятий,
    - есть фильтрация, поиск, сортировка;
- CRUD для мест проведения;
- CRUD для фотографий;

- Периодические задачи в Celery:
    - настоящий запрос погоды по АПИ сервиса для актуальных мероприятий;
    - проверка наступления времени публикации и автоматическая публикация;
    - автоматическая проверка и обновление превью;
    - рассылка email уведомлений;

- Эндпоинт для экспорта мероприятий в .xlsx файл (работает фильтрация);
- Эндпоинт для импорта мероприятий из .xlsx файла;

- Тесты АПИ для CRUD мероприятий;
- Подключен Swagger и описаны все эндпоинты;
- Логгирование ViewSet через LoggingMixin;

- Документация моделей данных в папке docs/.

---

## Инструкция по разворачиванию проекта:

Нужно клонировать репозиторий `git clone https://github.com/mark-ads/drf-first-one.git`.
Нужно создать .env по шаблону из .env.example (можно переименовать);

Развернуть можно в Docker или локально.

Docker:

```bash
docker-compose build    # собрать контейнеры
docker-compose up       # запустить контейнеры
docker-compose down -v  # остановить контейнеры
```
Если контейнер с PosgreSQL не успевает развернуться раньше контейнеров с проектом,
то нужно в файле .env увеличить показатель WAIT_PG_SEC до 10 или 20 секунд.

Локально (нужны развернутые Postgres и Redis):

1) внести настройки Postgres, Redis, а также DOCKER_RUN=False в .env;
1) создать виртуальное окружение `python -m venv .venv`;
2) активировать виртуальное окружение:
    - `.venv\Scripts\activate` (для Windows);
    - `source .venv/bin/activate` (для Linux);
3) установить зависимости `pip install -r requirements.txt`;
4) сделать миграции:
    - `python manage.py makemigrations`;
    - `python manage.py migrate`;
5) наполнить базу данных `python manage.py populate_db`;
6) запустить сервер `python manage.py runserver`.

Проект будет доступен по адресу:

http://localhost:8000/api/ (Browsable API)

или 

http://localhost:8000/api/docs/swagger/ (Swagger)

### Что дополнительно проверять при запуске:

- В папке sample_data лежит .xlsx файл, его можно импортировать через эндпоинт /api/events-import/;
    - При импорте часть строк пройдет валидацию и создадутся мероприятия и места;
    - Часть строк не пройдет валидацию и будут показаны ошибки в результатах.

- Можно сделать экспорт существующих мероприятий.
    - В /api/events/ выставить необходимые фильтры;
    - Вписать в URL между /events и ? /export. Должно получить URI вида /events/export?publish_date=...

- После запуска контейнеров будут созданы 2 мероприятия черновика, Celery проверит их и опубликует;
    - При публикации будут отправлены уведомления через email, EmailBackend получит эти письма и вывет в консоль.

- После запуска контейнеров будут созданы превью из фотографий для каждого мероприятия.

- Можно удалить соответствующую превью фотографию и тогда превью будет пересоздано Celery.
